基础（一）C语言发展简史与初探

Author: 万叶 Date: June 1, 2022  12:33:18 Category: C/C++

发展简史

编程语言发展到今天，很多语言由于其独特的优势还依旧受到大家的青睐，C语言就是其中之一。C语言是很多语言的“鼻祖”，因为从其他语言诸如Object-C、C++和C#等，都可以看到C语言的身影。

C语言诞生与1970年代初的贝尔实验室（Bell Labs）。Dennis Ritchie曾经发表过一篇名为《The Development of the C Language》的论文，该论文提到“C语言是在 1970 年代初设计的，作为新生Unix操作系统的系统实现语言。衍生自无类型语言BCPL，演化出类型结构；在微型机器上创建，作为改善微不足道的编程环境的工具”。贝尔实验室的工程师Ken Thompson希望能够为Unix平台开发一种编程语言，他修改了BCPL语言，然后创造了B语言。但是由于B语言的缓慢性和无法利用操作系统中的PDP-11功能，导致没有多少实用的程序使用B语言。后来，又由Dennis Ritchie在B语言的基础上，创造了C语言。

C语言的发展成为了Unix的基础。贝尔实验室的论文提到：“到1973年初，C语言的基本设计已经完成。C语言和编译器足够强大，可以让我们在今年夏天用C语言为PDP-11重写Unix内核。”如今，C语言已经真的很强大，可以在很多系统中使用到它。在1987年，Brian Kernighan和Dennis Ritchie共同编写了《The C Programming Language》，该书籍在正式标准成立之前一致被作为C语言的参考教材。五年后，美国国家标准协会（American Notional Standard Institute，简称ANSI）成立了X3J11委员会，建立了C语言的正式标准。

随后到了20世纪80年代，同样在贝尔实验室，Bjarne Stroustrup博士发明了C++语言。据说，他当时正在准备他的博士毕业论文，有机会使用一种叫做Simula的语言，这个语言主要是做仿真，但是由于Simula的语言执行效率低，实用性不强。然后，Bjarne Stroustrup博士就开始着手了“C with Classes”的研究工作。“C with Classes”是在C语言的基础上进行扩展的，属于C语言的超集。由于C语言的执行速度以及底层函数的性能不受程序移植的影响，Bjarne Stroustrup博士决定将面向对象的思想引入到C语言中。随后，C++不断增加新特性。虚函数（virtual function）、运算符重载（operator overloading）、多继承（multiple inheritance）、标准模板库（standard template library, STL）、异常处理（exception）、运行时类型信息（runtime type information）、名字空间（namespace）等概念逐渐纳入标准。

根据Toptal的说法，Unix操作系统基本上都是用C语言编写的，大多数的Linux系统也是用C语言编写，以及Oracle数据库、MySQL、MS SQL Server和PostgresSQL等数据库都有C语言参与的基础模块。C语言也是其他语言编译器或解释器的基础，例如Python、Perl。迄今为止，C语言任然是很多低级程序（如内核）中使用的通用语言。截止2022年4月，C语言的标准已经更新到了第四个标准——C17（也被成为C18）， 下一个版本的C2x标准，预计将在2022年12月1日完成。

初探C语言

C语言的特点非常明显，其优点是语言简洁、使用方便和灵活；运算符丰富、数据类型丰富，具有结构化的控制语句，是一门面向过程的编程语言；C语言允许直接访问物理地址，能进行位操作，同时能够集成汇编语言；生成目标的代码质量高，程序执行效率高。其缺点是写代码实现的周期长，可移植性差；代码编程自由，经验不足者容易出现越界、内存泄露等问题；对于平台库的依赖比较大。

C语言的应用是很广泛的，从网站的后台到底层操作系统，从多媒体应用到大型的网络游戏，都可以使用C语言来开发。在一些特定的领域，例如微处理器、嵌入式系统，以及各种系统的驱动等，C语言占据了半壁江山。

命名规范

C语言的命令规范通常应当遵守：

  1. 变量使用小写，两个单词之间使用下划线_分割；
  2. 常量通常大写；
  3. 只能使用字母、数字和下划线_，且不能以数字开头;
  4. 禁止使用关键字和系统函数命名。

类型限定符

类型限定符有extern、const、volatile、register，它们的含义是：

限定符 含义
extern 声明一个变量，extern声明的变量没有建立存储空间。
const 定义一个常量，常量的值不能修改。
Volatile 防止编译器优化代码。
register 定义寄存器变量，提高效率。register是建议型的指令，而不是命令型的指令，如果CPU有空闲寄存器，那么register就生效，如果没有空闲寄存器，那么register无效。

常量定义

常量有时候也被称为字面量。常量在程序执行期间不会改变，因此常量也可以称为只读变量。常量可以是C语言中任何基本数据类型。在C语言中，可以使用const来定义一个常量。

宏定义和const的区别

宏定义和使用const的两者区别是：

  1. 编译器处理方式不同。宏定义是在预处理阶段展开，const常量是在编译运行阶段使用；
  2. 类型和安全检查不同。#define 宏没有类型，不做任何类型检查，仅仅是展开。const 常量有具体的类型，在编译阶段会执行类型检查；
  3. 存储方式不同。#define 宏仅仅是展开，有多少地方使用，就展开多少次，不会分配内存。宏定义不分配内存，变量定义分配内存。const定义常量会分配内存。

在C语言中，变量的定义和变量声明是被区别开的。变量的定义用于为变量分配存储空间，还为变量指定初始值。程序中，变量有且仅有一个定义。变量声明用于向程序表明变量的类型和名字。变量的定义本身就属于变量的声明，但是使用了extern的声明不是变量的定义。

总结几点3：

  1. 带有初始化值的，不管是否有extern关键字，都被认为是定义；
  2. 凡是带有{}的函数一定是定义，否则就是声明；
  3. 除非有extern关键字，否则都是变量的定义。
